Why Responsive Classes Are Essential for Forms

Without responsive classes, a 3-column form on desktop stays 3-column on a tiny phone — making each input impossibly narrow. Responsive classes fix this by changing the width at each screen size.

The problem without responsive classes:

zc-col-4 alone = "always be one-third width, on EVERY screen"
On a 1400px desktop → each field is ~460px wide ✅ Great
On a 768px tablet → each field is ~250px wide ⚠️ Tight
On a 375px phone → each field is ~120px wide ❌ Broken — can't even type a name!

The fix — add responsive overrides:

zc-col-4 zc-col-md-6 zc-col-xs-12
On desktop → one-third width → 3 per row ✅
On tablet (≤992px) → md-6 kicks in → half width → 2 per row ✅
On phone (≤576px) → xs-12 kicks in → full width → 1 per row ✅

Simple rule of thumb:
• zc-col-12 → needs 0 responsive overrides (already full-width everywhere)
• zc-col-6 → needs 1 override: add zc-col-xs-12
• zc-col-4 or smaller → needs 2 overrides: add zc-col-md-6 zc-col-xs-12

The more columns on desktop, the more overrides you need.
1

1-Column Form

All fields full-width. No responsive classes needed — zc-col-12 is already 100% on every screen.

Live Preview
<div class="zc-row zc-gap-16">
  <div class="zc-col-12"> Full Name </div>
  <div class="zc-col-12"> Email </div>
  <div class="zc-col-12"> Message </div>
</div>
No responsive overrides needed. zc-col-12 = 100% width at every breakpoint. It literally can't get wider.
2

2-Column Form (Responsive)

Two per row on desktop → stacks to one-per-row on phones. Resize your browser to see it work!

Live Preview — Resize browser to test!
<div class="zc-row zc-gap-16">
  <div class="zc-col-6 zc-col-xs-12"> First Name </div>
  <div class="zc-col-6 zc-col-xs-12"> Last Name </div>
  <div class="zc-col-6 zc-col-xs-12"> Email </div>
  <div class="zc-col-6 zc-col-xs-12"> Phone </div>
</div>
Desktop zc-col-6 → half width → 2 per row
Phone ≤576px zc-col-xs-12 kicks in → full width → 1 per row

Why no md class? Because half-width is still comfortable on tablets (~480px per field). You only need to stack on small phones.
3

3-Column Form (Responsive)

Three per row → two on tablet → one on phone. This is where responsive classes become critical — without them, each field would be ~120px on a phone.

Live Preview — Resize browser to see 3→2→1 flow!
<div class="zc-row zc-gap-16">
  <div class="zc-col-4 zc-col-md-6 zc-col-xs-12"> First Name </div>
  <div class="zc-col-4 zc-col-md-6 zc-col-xs-12"> Middle Name </div>
  <div class="zc-col-4 zc-col-md-6 zc-col-xs-12"> Last Name </div>
  <!-- same pattern for remaining fields -->
</div>
Desktop >992px zc-col-4 → one-third → 3 per row
Tablet ≤992px zc-col-md-6 → half → 2 per row
Phone ≤576px zc-col-xs-12 → full → 1 per row

Only the 3-column section needs responsive classes. The flat zc-col-12 section is already full-width everywhere — no overrides needed. You only add responsive classes to fields narrower than full-width.

Responsive overrides per section type:
3-col fields → zc-col-4 zc-col-md-6 zc-col-xs-12 (2 overrides)
2-col fields → zc-col-6 zc-col-xs-12 (1 override)
1-col fields → zc-col-12 (0 overrides)
7

Inline Label + Field (Responsive)

Label sits beside the field on desktop → stacks above it on small screens using zc-col-sm-12.

Live Preview
<div class="zc-row zc-gap-12 zc-align-items-center">
  <label class="zc-col-3 zc-col-sm-12 zc-no-shrink">Company</label>
  <div class="zc-col-common">
    <input type="text" />
  </div>
</div>
Desktop zc-col-3 on label → sits beside the input (25% width)
≤768px zc-col-sm-12 → label goes full-width, input wraps below it

Why zc-row not zc-flex? Because zc-row wraps items. When the label becomes sm-12 (full-width), the input wraps to the next line. With zc-flex (no wrap), items would squeeze instead.
8

Uneven Widths (Responsive)

Mixed widths (8+4, 5+4+3, 3+9) that equalize to halves on tablet and stack on phone.

Live Preview — Resize to see equalization!
<!-- Desktop: uneven (8+4, 5+4+3, 3+9) → Tablet: equal halves → Phone: stacked -->

<div class="zc-row zc-gap-16">
  <div class="zc-col-8 zc-col-md-6 zc-col-xs-12"> Street </div>
  <div class="zc-col-4 zc-col-md-6 zc-col-xs-12"> Pin Code </div>

  <div class="zc-col-5 zc-col-md-6 zc-col-xs-12"> City </div>
  <div class="zc-col-4 zc-col-md-6 zc-col-xs-12"> State </div>
  <div class="zc-col-3 zc-col-md-6 zc-col-xs-12"> Country </div>

  <div class="zc-col-3 zc-col-md-6 zc-col-xs-12"> Code </div>
  <div class="zc-col-9 zc-col-md-6 zc-col-xs-12"> Phone </div>
</div>
The equalization trick: On desktop, fields have unequal widths (8+4, 5+4+3). On tablet, everything normalizes to zc-col-md-6 (equal halves) for a clean look. On phone, everything stacks with zc-col-xs-12. The 3-item row (5+4+3) wraps as 2+1 on tablet — totally fine, the third field just flows down.
