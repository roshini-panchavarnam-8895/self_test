Your first 5 minutes with AI

Never used an AI tool? Pick any chat — Claude (claude.ai), ChatGPT (chatgpt.com), or Gemini (gemini.google.com). Try these 5 steps right now:

1Open any AI chat — all are free to start. Just pick one and type.
2Introduce yourself: "I'm a frontend dev working on a SCSS design system" — this gives AI context so answers are relevant to you.
3Ask with details: "How should I structure color tokens for light and dark themes in SCSS?" — more specific = better answer.
4Talk back: Say "Show me code" or "Make it simpler" — AI gets better the more you guide it.
5Read before using: AI is fast but not always right. Review, test, understand — then use.
The #1 rule — more important than any tool or prompt

Before any task — easy or tough — stop and ask:
"How can AI help me do this?"

This is the single most important habit. Before you start any work — a bug fix, a component, a feature — pause 10 seconds and think: "Can AI help me here?" Most people only think of AI when stuck. The ones who get 10x value think of it before they start.

⚡ This is vibe coding.

You describe what you want in plain language. AI builds the first version. You refine, guide, iterate. Instead of writing every line, you direct AI like a creative partner — focusing on what to build, not how to type it. That's the shift. That's the skill. That's what this guide is about.

New code? → AI scaffolds first Error? → Paste to AI before googling Review? → AI scans first Planning? → Brainstorm with AI Repetitive? → AI automates it Learning? → AI explains it

Understand the building blocks

Key concepts you'll hear about

You don't need to memorize — just understand what they mean so you're not lost when someone mentions them.

A
Agent
A worker who plans and acts on their own

A regular AI answers questions. An agent actually does things — reads files, runs commands, creates folders. You give it a goal, it figures out the steps. Claude Code and Cursor Composer are agents.

Reads your code→Plans changes→Creates files→Updates components→Tests
SA
Sub-agent
Helper the main agent delegates to

For complex tasks, agents spawn sub-agents — focused workers for specific parts. One scans files, one maps tokens, one applies fixes — all in parallel.

S
Skills
Instruction manuals for AI

Rules and instructions that teach AI a specific task. Write your conventions once as a skill, AI follows them automatically — no repeating "use BEM with .zc- prefix" every time.

MD
SKILL.md & config files
Cheat sheet you give AI before it works

Different tools read different files, same purpose — giving AI your project context automatically.

SKILL.md
Claude Code/Chat. Rules for specific tasks.
CLAUDE.md
Project-level rules. Auto-read every session.
.cursorrules
Same idea for Cursor IDE.
copilot-instructions.md
GitHub Copilot's version.
M
MCP (Model Context Protocol)
Plug-ins connecting AI to other tools

Lets AI connect to Figma, Google Drive, Slack, Jira directly. Instead of copy-pasting between apps, AI reads and writes to services for you.

CW
Context window & Artifacts
AI's short-term memory and scratchpad

Context window = how much AI can "see" at once. Long chats lose old context. Artifacts = outputs AI creates (HTML, React, docs) that preview in a side panel.

All these connect: SKILL.md → agent reads it → spawns sub-agents → connects to tools via MCP → produces artifacts. That's modern AI-assisted development.

How to actually use it

4 rules that change everything

You've seen the #1 rule above. These four rules are how you apply it daily.

01

Talk to it like a new team member

Don't say "fix the bug." Say "there's a bug in the dropdown — it doesn't close on outside click. Here's my useEffect: [paste]."

VagueFix my CSS
ClearMy dropdown stays open when clicking outside. Using React + useRef. Here's my code: [paste]. What's wrong?
02

Have a conversation, not a one-shot

The first answer is a draft. Say "now add validation" or "make it RTL" or "that's not what I meant."

One-shotBuild login page → copy → done
IterativeBuild form → add validation → make RTL → match tokens → add loading
03

Give context upfront

AI doesn't know your project. 30 seconds of context saves 10 minutes of fixing irrelevant answers.

No contextWrite a button component
With contextButton for Zoho Creator design system. SCSS, BEM, $zc- tokens, RTL, sm/md/lg.
04

Trust but verify — always

AI is confident even when wrong. Read the code, understand the logic, test locally, then use it.

BlindCopy AI code → push to production
SmartRead it → understand it → test it → then use it

6 prompt formulas that always work

Don't start from blank. Pick a formula, fill in your details.

1
The context-first formula
▾

Start every prompt with context. "Who am I, what am I working on, what do I need?"

I'm working on [project] using [stack]. I have [what exists]. I need [task]. Constraints: [rules]
Example
I'm building a component library for Zoho Creator using SCSS with BEM. I have button and input done. I need a dropdown with RTL, sm/md/lg sizes, matching the button pattern. Transitions under 200ms.
2
The "act as" formula
▾

Give AI a role for the right depth and perspective.

Act as a [role] specializing in [domain]. Review/Create [thing]. Focus on [aspects].
Example
Act as a senior frontend dev specializing in RTL layouts. Review this CSS and list every property needing RTL handling. Show current code and corrected version.
3
The "show me options" formula
▾

Not sure about the approach? Ask for 2-3 options with trade-offs.

I need to [task]. Show me [2-3] approaches. For each: how it works, pros/cons, when to use. My setup: [constraints]
Example
I need to handle theming in SCSS. Show me 3 approaches — CSS custom properties, SCSS maps with mixins, data-attributes. Show a button example for each. Which works best for RTL + multi-brand?
4
The "transform this" formula
▾

Input → output with rules. Best for conversions and refactoring.

Here is [input]. Transform into [output format]. Rules: [requirements]
Example
Here's a screenshot of a Creator PageBuilder form. Transform to ZML markup. Use zc-form container, map fields to ZML types, keep two-column grid, add matching placeholders.
5
The "review & fix" formula
▾

Paste code, get structured review. Great for learning.

Review this [code]: [paste] Check for: [aspects] For each issue: show line, explain, show fix.
Example
Review this SCSS. Check for: inconsistent tokens (should use $zc-), missing RTL (use logical properties), hardcoded values. For each: show line, explain, give fix.
6
The "step by step" formula
▾

For complex tasks, ask AI to plan before coding.

I need to [complex task]. Before code: 1) List steps 2) Identify issues 3) Implement step by step
Example
Add RTL support to our dropdown. Before code: list all CSS needing RTL handling, identify JS logic assuming LTR, note icons needing mirroring. Then implement each fix with explanations.
Quick rules

Do's and don'ts

✓ Do this

✓Give context — project, stack, conventions
✓Iterate — ask follow-ups, say "make it better"
✓Paste actual code, not vague descriptions
✓Ask "why?" — learn from AI, don't just copy
✓Read and test every line before using
✓Save prompts that worked — reuse them
✓Break big tasks into smaller prompts

✗ Don't do this

✗Copy AI code to production without reading
✗Use vague one-word prompts
✗Share passwords, API keys, or secrets
✗Assume the first answer is always right
✗Dump 10 things in one prompt
✗Give up after one bad answer — rephrase
✗Use AI as a crutch — use as a multiplier

Learn from others

5 mistakes everyone makes at first

01Being too vague
"Fix my code" — AI doesn't know your project.
Fix: Say what's broken, what you expected, and paste your code. One good prompt beats ten vague ones.
02Accepting everything without reading
AI writes confident code with subtle bugs.
Fix: Read every line. Ask "explain this part" if unsure. Test locally. Never push code you can't explain.
03Using the wrong tool
Using chat for coding, or Cursor for quick questions.
Fix: Quick question → Chat. Browser debugging → Extension. Multi-file → Cursor. Repo-wide → Claude Code.
04One giant prompt for everything
10 requirements in one message = nothing done well.
Fix: One focus per message. Structure first, then styling, then interactions.
05Starting new chat every time
Losing all context AI built about your project.
Fix: Keep one chat for related tasks. AI remembers everything in the current conversation.
😄

Every mistake above? We've all made them. The person who wrote this guide made every single one. The difference between a beginner and someone good at AI is just 2-3 weeks of practice. That's it. You're closer than you think.
