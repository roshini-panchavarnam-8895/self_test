Stream 1
Design Flow
From PRD to approved design. AI generates, humans refine and approve.
1
PRD / Prompt → AI Design Generation AI
The PRD document or a consolidated design prompt is fed into an AI design tool. It produces initial screens, component layouts, token values, and specs covering both builder UI and live views.
ToolFigma AI, Galileo AI, or custom prompt-to-design agent
InputPRD document or structured design prompt with feature requirements
OutputFigma screens with component usage, token references, layout structure
2
Manual Refinement Human
Designer reviews AI output and makes minimal corrections — spacing tweaks, visual hierarchy, edge cases, brand tone alignment. Goal is refinement, not recreation.
OwnerUI/UX Designer
3
Guideline & WCAG Review AI Plugin
AI plugin runs automated checks: design system adherence, WCAG AA/AAA contrast, touch targets, focus order, RTL compatibility. Outputs a structured report with severity levels and fix suggestions.
ToolStark / Axe for Figma / Include — or custom plugin
OutputCompliance report: pass/fail per rule, severity, suggested fixes
4
New Component / Token Push Automated
If new components or tokens are introduced, they're pushed to the Git repo. This triggers auto-update to the cloud component library, keeping design and dev in sync.
PipelineFigma Tokens → Style Dictionary → Git commit → CI publish
5
PM & Development Review Gate Human
PM and dev lead sign off. Only after this gate does the design enter the development pipeline. Clear handoff with Figma links, annotated specs, and token references.
OwnerProduct Manager + Dev Lead
Stream 2
Development & Review Flow
From design handoff to merged MR. AI assists at pre-check, generation, testing, and review.
1
Design Pre-check AI Platform
Before any code is written, design URL (or screenshots / PRD) is fed into a component-trained AI platform. It validates: are library components used correctly? WCAG met? Tokens consistent? Outputs a compliance report.
ToolCustom AI platform with RAG over component library + guidelines
InputFigma URL, screenshots, or PRD document
OutputStructured compliance report (errors, warnings, info) with fix suggestions
2
Code Generation AI Code Gen
If pre-check passes, AI generates HTML. Smart reuse: if CSS already exists in the component library → no regeneration. For new components → creates full file set (HTML + CSS + JS) following team conventions.
ToolLLM (Claude/GPT) + RAG over component library + code standards
ContextComponent docs, CSS tokens, folder structure, naming conventions, examples
3
UI Sanity Review Human
UI developer does a visual sanity check — comparing generated output against approved design. Checks edge cases, responsive behavior, RTL layout correctness.
OwnerUI Developer
4
Engineering Integration Human
Engineering team wires UI code with business logic — APIs, state management, event handling, backend services.
OwnerEngineering Team
5
Final Sanity Review Human
Post-integration review: spacing, animations, responsive behavior, data-driven states, RTL compliance all verified.
6
AI Browser + Unit Testing AI Extension
AI browser extension runs cross-browser visual testing (screenshot diffs) and auto-generates unit tests for UI components. Catches rendering inconsistencies and layout shifts.
ToolCustom AI browser extension + Playwright/Puppeteer backend
OutputVisual diff report + generated test files
7
Code Review & MR AI Review Human
AI first pass: code quality, performance, security, style compliance. Human final pass: reviewer has ultimate authority, adds MR labels, approves merge.
AI ToolCodeRabbit / Sourcery / Custom (Claude API integration)
HumanSenior dev / Tech lead — final authority on MR labels and merge
Stream 3
Component Library Sync Flow
The bridge between design decisions and developer tooling — automated and version-controlled.
1
Dual-Format Component Library Foundation
Maintain a component library in both design (Figma) and code (HTML/CSS/JS) with identical naming. Covers builder UI + live views. Single source of truth.
2
Auto-Sync: Design → Cloud Library Automated
Automated pipeline syncs design changes to cloud dev library. Review labels (new, modified, deprecated, breaking) auto-applied via diff analysis.
PipelineFigma API webhook → Diff engine → Auto-PR with labels → CI publish on merge
3
Cloud Library → Code Extensions Auto / Manual
Changes propagate to VS Code extension and Chrome drag-drop extension via cloud push or manual trigger. Each update carries the review label.
MechanismVersion manifest on CDN → Extension checks on startup → Pulls if newer
Requirements
What's Needed to Make This Work
Every prerequisite organized by priority. Critical items must exist before any AI step functions.

Component Library — Design

Critical
  • Figma components with variants for every UI element (builder + live views)
  • Design tokens defined (colors, spacing, type, elevation, radius) via Figma Tokens
  • Naming convention aligned 1:1 between Figma and code
  • RTL variants with mirroring specs for Arabic support
  • Versioning at the component level (semantic versioning)

Component Library — Code

Critical
  • Coded components (HTML/CSS/JS) matching every Figma component 1:1
  • CSS token file auto-generated from design tokens via Style Dictionary
  • Component docs — props, states, usage guidelines, do/don't examples
  • Published via internal npm registry or CDN for consumption
  • Storybook or equivalent for visual catalog and regression testing

AI Context Layer

Critical
  • Structured docs — .meta.json per component (props, variants, usage as data)
  • Guidelines as data — JSON/YAML format, not just PDF
  • WCAG as assertions — machine-checkable conditions, not prose
  • Coding standards — naming, folder structure, CSS methodology, JS patterns
  • Example mappings — 5-10 complete PRD → design → code as few-shot refs

Design → Code Sync Pipeline

High
  • Figma API integration to detect component/token changes
  • Token transformation — Style Dictionary (tokens → CSS vars → JSON)
  • Diff engine — classify: new / modified / deprecated / breaking
  • Auto-PR with review labels on Git commit
  • CI/CD to publish updated library on merge

Developer Extensions

High
  • VS Code extension for component snippet insertion
  • Chrome extension (drag-drop library) connected to cloud source
  • Auto-update — manifest check on startup, pull if newer
  • Component search and preview within editor context

AI Tooling Stack

Medium
  • Design gen: Figma AI / custom agent for prompt-to-design
  • WCAG plugin: Stark / Axe / Include for accessibility checks
  • Pre-check: Custom LLM + RAG over component library
  • Code gen: LLM + RAG trained on components + standards
  • Testing: AI extension for visual regression + unit test gen
  • Code review: CodeRabbit / Sourcery / Custom on Git

Process & Governance

Medium
  • PRD template optimized for AI consumption
  • Design handoff checklist — tokens, states, RTL verified
  • MR labeling convention aligned across AI + human reviewers
  • Coverage gate in CI (% UI from library vs custom)
  • Feedback loop — tag bugs on AI code, track accuracy

Metrics & Tracking

Medium
  • AI accuracy: % generated code shipped unchanged
  • Library coverage: % UI from library vs custom
  • Time-to-ship: design-to-production cycle reduction
  • Review efficiency: AI suggestion acceptance rate
  • Bug attribution: AI-generated vs human-written bugs

Core Principle

The strongest version of this process isn't one where AI does the most work. It's one where AI has the best context to work with, and humans make the decisions that matter. Build the foundation (component library + AI context layer) first — everything else plugs into it.
